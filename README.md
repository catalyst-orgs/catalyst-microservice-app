# Catalyst Microservice App

Welcome to the Catalyst Microservice App! This repository is a production-ready FastAPI microservice template, designed for seamless onboarding into the Project Catalyst Internal Developer Platform (IDP).

## Features
- **FastAPI**: High-performance Python web framework for building APIs
- **Kubernetes Native**: All manifests for deployment, scaling, and ingress are included
- **ArgoCD GitOps**: Automated deployment and continuous delivery via ArgoCD
- **Istio Ambient Mesh**: Out-of-the-box service mesh integration for secure, observable, and resilient traffic
- **Backstage Integration**: Catalog-ready with techdocs and metadata for developer portals
- **CI/CD Ready**: Easily connect to your preferred CI pipeline

## Getting Started

### 1. Prerequisites
- Kubernetes cluster (EKS, GKE, AKS, or local)
- ArgoCD installed and configured
- Istio Ambient Mesh enabled
- Docker & Python 3.11+

### 2. Local Development
```bash
# Clone the repo
 git clone https://github.com/catalyst-orgs/catalyst-microservice-app.git
 cd catalyst-microservice-app

# Start backend (FastAPI)
 cd backend
 pip install -r requirements.txt
 uvicorn main:app --reload

# Start frontend (optional)
 cd ../frontend
 docker build -t catalyst-frontend .
 docker run -p 8080:80 catalyst-frontend
```

### 3. Deployment via ArgoCD
- All Kubernetes manifests are in the `kubernetes/` directory.
- The `argocd-application.yaml` defines the ArgoCD Application resource.
- On merge to `main`, ArgoCD will auto-sync and deploy your service.

### 4. Accessing the Service
- The service will be available at the domain you specified during onboarding (see Ingress manifest).
- Example: `https://api.platformcatalyst.dev`

## Project Structure
```
backend/    # FastAPI app
frontend/   # Optional dashboard UI

```

## Observability & Operations
- **Metrics**: `/metrics` endpoint for Prometheus
- **Health**: `/health` and `/ready` endpoints
- **Istio**: All traffic is routed through Istio Ambient Mesh for security and observability
